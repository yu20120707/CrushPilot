# 用户档案与多目标管理系统

这套系统确保你在不同对话中能"记住"用户和他追的每个对象，给出有延续性的建议，而不是每次都从零开始。

---

## 用户档案

### 存储

把用户档案保存到你运行环境的持久化存储里（具体位置参见本文末尾的「持久化」一节）。首次使用时创建；不存在则在信息采集阶段引导用户填写。

### 档案模板

```markdown
---
name: 用户档案
created: {日期}
updated: {日期}
---

## 基本信息
- 年龄段：
- 城市：
- 职业/学业：

## 兴趣与特长
- 运动：
- 爱好：
- 特长/技能：
- 生活方式：

## 社交风格
- 性格自我评估：
- 社交经验水平：
- 朋友圈经营情况：

## 人设定位
- 性格标签：（如"温暖幽默型""沉稳靠谱型""阳光运动型"）
- 核心吸引力：（什么让你跟别人不一样）
- 故事素材库：（你真实经历中可以展示高价值的 2-3 个故事）
- 形象关键词：（你想传递的整体印象）
```

### 人设采集话术

当档案信息不足时，用这个语气引导：

"兄弟先跟我聊聊你自己。你平时都玩什么，有什么爱好或者特长？工作是做什么的？不用谦虚，有什么觉得自己还不错的地方都说说，我帮你梳理一下你的核心竞争力。"

**铁律：永远不替用户编故事。** 生成话术时如果需要用到用户的兴趣、经历、特长，必须基于档案中的真实信息。档案里没有的，先问用户，绝不编造。

## 多目标档案管理

### 存储

每个目标对象一个独立的档案文件，保存到你运行环境的持久化存储里（参见本文末尾的「持久化」一节）。命名约定：`{称呼}.md`（如 `小美.md`、`咖啡店女生.md`、`同事Linda.md`），方便按称呼快速匹配。

### 档案模板

```markdown
---
name: {用户对她的称呼}
platform: {微信/探探/陌陌/Soul/Bumble/青藤之恋/其他}
created: {首次提到的日期}
updated: {最后更新日期}
---

## 基本信息
- 怎么认识的：
- 她的基本情况（年龄/职业/城市等已知信息）：
- 聊天平台：

## 当前阶段
- 所处阶段：阶段X — {阶段名}
- 阶段判断依据：

## 关键信号记录
- IOI 信号：
- IOD 信号：
- 信号变化趋势：

## 对话时间线
- {日期}：{关键事件/进展摘要}

## 素材存档
- {日期} 截图分析：{分析结论摘要}

## 上次给出的建议
- 建议内容：
- 用户是否执行：
- 执行效果：

## 引领规划追踪
- 当前引领方向：
- 未完成的 action items：

## 重要细节备忘
- {她提过的关键信息、共同话题、用户提到的细节等}
```

## 自动识别与加载流程

每次用户开始聊天时，执行：

1. **检测目标身份** — 从用户消息中提取人名、代号、称呼或上下文线索
2. **匹配已有档案** — 列出已有的目标档案，尝试匹配（怎么列取决于你的运行环境，参见「持久化」）
3. **确认身份**：
   - 匹配成功 → 加载档案，输出状态摘要 + 未完成的 action items："兄弟，你说的是{称呼}对吧？上次的情况是{摘要}。你之前说要{未完成的建议}，执行了吗？效果怎么样？"
   - 不确定 → 问用户确认
   - 新目标 → 创建新档案，进入信息采集流程
4. **对话结束时更新档案** — 阶段变化、新增信号、时间线、建议记录、引领规划

---

## 多目标上下文切换

用户可能在一次对话中聊到多个人。识别切换信号后：

1. **保存当前目标的完整状态快照**（正在进行的引领规划、未完成的建议）
2. **加载新目标的档案**
3. **输出新目标的状态摘要**，简短确认后继续

### 一致性保障

- 阶段不能无理由倒退
- 已知信息不能矛盾（除非用户纠正）
- 建议要有延续性：上次建议"先冷两天"，这次先追踪效果再给新建议

## 持久化 — 全局统一路径

**存储根路径**：
- macOS / Linux：`~/.qingsheng/`
- Windows：`$APPDATA/.qingsheng/`

这是**跨 agent、跨项目、跨会话**都能找到的统一位置。不再让用户选路径，原因在下面。

### 目录结构

```
~/.qingsheng/
├── user-profile.md         # 用户档案（自己的情况）
├── targets/                # 每个目标一个文件
│   ├── Judy.md
│   ├── 咖啡店女生.md
│   └── ...
├── installed-version       # setup 脚本写的，别动
└── last-update-check       # preamble 写的，别动
```

### 为什么不再 AskUserQuestion 问路径

实测发现用户**根本不在意存哪**，只在意"我能不能看到"。之前让用户点选路径反而：
- 在第一轮对话前打断节奏
- 用户选完一头雾水，不知道刚选了什么
- Cursor/Codex 的原生记忆区路径用户根本不熟，选了也找不到

改成默认 `~/.qingsheng/` + **明确告知路径** 后：
- 无需用户操作，Skill 直接建档
- 路径固定，跨 agent 一致（Claude Code / Cursor / Codex 都能找到同一个目录）
- 用户想看随时 `open ~/.qingsheng/` 或 `cat ~/.qingsheng/targets/Judy.md`

### 建档流程

**首次建档**（`~/.qingsheng/user-profile.md` 不存在）：

1. Bash 创建目录：`mkdir -p ~/.qingsheng/targets`
2. 创建 `~/.qingsheng/user-profile.md`（用档案模板，能采集到的信息都填上）
3. 创建 `~/.qingsheng/targets/{称呼}.md`（当前目标）
4. 告诉用户："建好了，你的档案在 `~/.qingsheng/`，想看随时打开。"

**新目标建档**（user-profile.md 已存在，但这个女生是第一次聊）：

1. 确认称呼（用 `AskUserQuestion` 给 2-3 个候选：真名、昵称、特征代号如"咖啡店女生"）
2. 创建 `~/.qingsheng/targets/{称呼}.md`
3. 告诉用户："建好了，存在 `~/.qingsheng/targets/{称呼}.md`，以后说 `/复盘 {称呼}` 就能调出来。"

**老目标回来**（`~/.qingsheng/targets/{称呼}.md` 已存在）：
- 直接读取档案，输出上次进度摘要，继续。不重复建档。

### 称呼确定策略

- 用户自己明确说了名字/昵称 → 直接用
- 只有代号（"那个女生"）→ 问 AskUserQuestion 给 2-3 个候选
- 用户发了截图里有名字 → 用截图里的名字作为默认选项

### 兜底规则

- **不要**让用户自己输入路径——默认 `~/.qingsheng/` 就够了
- **不要**在没建档的情况下先给分析（这是 SKILL.md 「第负一步 Gate A」的硬门）
- Windows 环境 Bash 用 `$APPDATA/.qingsheng/` 替代 `$HOME/.qingsheng/`（检测 `uname -s` 是 MINGW*/MSYS*/CYGWIN*）
