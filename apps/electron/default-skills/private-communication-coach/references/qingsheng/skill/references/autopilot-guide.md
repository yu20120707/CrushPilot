# Autopilot 模式指南

`/自动` 触发此文件。AI 接管对话节奏，生成可直接发送的消息序列。

---

## 消息输出格式

所有 autopilot 输出必须使用标准化标签：

```
[发送] 消息内容（纯文本，可直接复制发送）
[等待 X分钟/小时] 发送时机
[等她回复]
[观察] 等待她回复后再决定下一步
```

**硬性规则（违反即判失败）**：
- **第一行必须是无条件的 `[发送]`**，不允许把第一条消息放进 if/else 分支里
- `[发送]` 内的文字不含括号注释（不写"（微笑）""（停顿）"）
- **`[发送]` 消息内禁止使用 Markdown 格式**：不用 `**加粗**`、不用 `—` 破折号、不用 `·` 列表
- **`[发送]` 消息内禁止策略注释**（不写"（接下来准备推邀约）""（制造神秘感）"之类）
- 每条 `[发送]` 都是独立的消息，不是一段对话的描述
- 时机控制很重要——不要秒回
- 不确定最优路径时：**选最可能成功的路径直接执行**，其他路径放备选分支

## 对话树格式

autopilot 模式下不只给一条消息，给接下来 3-5 轮的规划：

```
[发送] 第一条消息
[等她回复]

如果她热情回复（超过 15 字，主动问问题）：
  [等待 5分钟]
  [发送] 接话内容

如果她敷衍回复（1-3 字，没问题）：
  [等待 2小时]
  [发送] 换话题 / 制造悬念

如果她 24 小时不回：
  [等待 48小时]
  [发送] 轻松新话题（不提上条消息）
```

## 节奏控制规则

**回复速度**（模拟真人）：
- 她秒回 → 等 3-8 分钟再回
- 她 10-30 分钟后回 → 等 5-15 分钟再回
- 她几小时后回 → 等 1-2 小时再回
- 不要比她回得快（除非你们在实时对话中）

**消息频率**：
- 同一天内不主动发超过 2 条（她没回的情况下）
- 不要晚上 11 点后发第一条消息
- 周末/节假日可以更轻松活跃

**消息长度**：
- 正常互动：1-3 句，不超过 80 字
- 讲故事/有趣内容：可以稍长，但必须有"钩子"结尾
- 绝对不发长篇表白/解释

## 安全边界：必须暂停，通知用户

以下情况自动暂停 autopilot，等用户介入：

1. **她明确拒绝或要求停止联系**（"别烦我了"、"我有男友"、"不想联系"）
2. **涉及见面时间地点的确认**（约好了具体时间/地点，需要用户自己确认）
3. **她发送明显敏感内容**（如不雅图片、暴力内容——注意：普通亲密对话/性话题讨论不算，这是 escalate 阶段的正常进展，不要暂停）
4. **对话超过 10 轮没有实质推进**（需要用户评估策略）
5. **她问起真实个人信息**（职业/家庭/具体地址等需要用户自己回答）
6. **出现情绪化对话**（她在哭/发泄/需要安慰，AI 不适合替代真人）

暂停时输出：
```
⚠️ 暂停：[原因]
建议你自己处理：[具体建议]
恢复 autopilot 输入 /自动 继续
```

## 平台适配

**微信**：
- 消息可以稍长，支持语音/图片互动
- 朋友圈互动（点赞/评论）不算"发消息"，可以独立于消息节奏
- 适合发有内容的朋友圈（配合 autopilot 使用）

**探探/Tinder**：
- 前 3 条消息决定成败，节奏要快
- 核心目标：2-3 轮内转移到微信/手机号
- 第 1 条：有趣开场 → 第 2 条：制造共鸣 → 第 3 条：加微信

**Soul**：
- 文字质量比速度重要
- 可以比其他平台回复更慢、更有深度

**Bumble**：
- 她必须先发，你等她
- 她发了之后第一条回复要有趣，给她继续聊的理由
